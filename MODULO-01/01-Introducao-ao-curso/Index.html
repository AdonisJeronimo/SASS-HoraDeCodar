MÓDULO 01

O que é um pré-processador?

  • O pré-processador é um software que recebe uma entrada de dados e gera uma saída;
  • Geralmente modificando os dados de ma forma que possa ser aceita por outro software;
  • O SASS é um pré-processador de CSS;
  • Ele recebe funcionalidades avançadas que naõ são entendidas pelo CSS em si, mas é processado e transformado em CSS puro.

O que é SASS

  • O SASS é um pré-processador de CSS que possui muitas funcionalidades que irão aumentar a nossa produtividade de código para o desenvolvimento web, oferecendo diversos recurso como variaveis, mixins, nesting, herança e muito mais. Varios projetos grandes no mercado utilizam o SASS, como o Bootstra. Com isso, entende-se que, com o SASS será possível escrevermos menos códigos e ainda sim, gerarmos um CSS de qualidade.

Alguns recursos do SASS

  • Pré-processamento: geração de código CSS otimizado e válido para os navegadores mais atuais;
  • Variáveis: reaproveitamento de estilos em todo o projeto, resultando em uma facilidade maior de manutenção;
  • Nesting: encadeamento de seletores de CSS;
  • Partials: Divisão de responsabilidades no CSS;
  • Mixins: especie de função para reaproveitamento de estilo
  • E muito mais.

Diferença entre SASS, SCSS e CSS

  • O SASS e o SCSS são bem parecidos, porém o SASS possui mais recursos e tem uma sintaxe mais simples;
  • Com o SASS é possivel pré-processar SCSS, ou seja, podemos escrever os dois;
  • Ambos tem o mesmo papel que é trazer novas funcionalidades ao CSS;
  • No fim, todo o código gerado com ambos viram CSS puro, sendo assim, aceito pelos navegadores;
  • É possivel que futuramente essas funcionalidades de SASS e SCSS sejam incluidas diretamente no CSS.

Instalação de dependencias

Para criar os projetos e rodar o SASS, precisamos apenas de Node.js e o npm. Obviamente, também é preciso usarmos um editor de código (que no meu caso é o VSCode).

Para instalar o Node.js, entraremos no site oficial e iremos baixar a versão mais recente LTS.

Instalando SASS

  • Vamos instalar o SASS com o Node.js (npm) de forma global;
  • Para isso, utilizaremos o seguinte comando;
  • npm install -g sass
  • Para termos certeza que a instalação do SASS foi efetuada de forma correta, execute o comando:
  • sass --version
  • Isso resultará na versão do SASS instalada na máquina;
  • Agora podemos executar o SASS em qualquer projeto.

Instalando extensões SASS no VS Code

  • Para melhorar nossa experiência de programação em SASS, vamos instalar uma extensão no VSCode
  • O nome dela é SASS e está disponível gratuitamente no marketplace;
  • Teremos funcionalidades como: auto-complete, highloght de código, formatação e muito mais.

Primeiro Projeto SASS

  • Crie uma pasta;
  • Crie a pasta sass e css;
  • Crie o arquivo index.html (para consumir o CSS gerado pelo SASS);
  • Após gerarmos um código SASS, precisamos buildar o código para CSS e para isso, abriremos o terminal na pasta e colocaremos o seguinte comando:
  • sass sass/style.sass css/styles.css
  • Temos que sass/style.sass é a pasta de origem do arquivo SASS e css/styles.css é a pasta de saida onde o código SASS sera transformado em CSS puro;
  • Para que não seja necessario buildarmos o código toda vez que fizermos uma alteração no SASS, podemos rodar o seguinte comando:
  • sass —watch sass/style.sass:css/style.css
  • Com isso, toda vez que fizermos alguma modificação no SASS automaticamente será atualizado no CSS e no navegador(como o Live Server faz).

Como debugar código com o SASS

O debug de SASS é muito simples, se errarmos algo no código o CSS não vai compilar corretamente, assim o Sass irá imprimir um texto no HTML com informações de onde erramos, para identificaremos o problema, corrgirmos e compilarmos novamente.
