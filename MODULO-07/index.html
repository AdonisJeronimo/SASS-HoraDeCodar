Módulo 7

Números

  • Os números em SASS são compostos por duas partes: o NÚMERO e a UNIDADE.
  • EXEMPLO: 100% - 100 NÚMERO - % UNIDADE.
  • Algumas unidades são INCOMPATÍVEIS, como REM e PX.
  • A precisão de números vai até 10 CASAS.

Strings

  • String são os TEXTOS que escrevemos.
  • As strings são inseridas ENTRE ASPAS e também SEM ASPAS.
  • "Roboto"(com aspas), hidden (sem aspas).
  • Podemos INTERPOLAR valores em string.
  • Podemos CONCATENAR strings com +.

Cores

  • A cor é considera um tipo de dado em SASS
  • Podemos inserir em HEXADECIMAL, NOME DA COR, RBG, HSL.
  • Todos estes formatos também são aceitos em CSS.

Lista

No SASS, as listas são estruturas de dados que permitem agrupar vários valores relacionados em uma única entidade (são como os arrays). Uma lista é uma sequência ordenada de valores separados por vírgulas.

As listas podem conter diferentes tipos de valores, como números, strings, variáveis ou até mesmo outras listas. Elas são usadas para armazenar conjuntos de valores relacionados, como uma lista de cores, uma lista de tamanhos de fonte, uma lista de propriedades CSS, entre outros.

As listas no SASS oferecem algumas funcionalidades e recursos úteis. Você pode acessar elementos específicos da lista usando a notação de índice, onde o primeiro elemento tem o índice 1. Por exemplo, para acessar o segundo valor da lista $colors, você usaria $colors[2].

Maps

  • SINTAXE: $map: ("chave":1,"teste":50)

Booleans

  • Os booleanos são representados por TRUE e FALSE.
  • Toda expressao lógica retonar um verdadeiro ou falso.
  • Por exemplo: 10 > 2 (true)
  • Em if’s no SASS podemos obvservar a utilização dos booleanos.

Igualdade e Diferença

  • Caso haja a necessidade de comparar valores, se são iguais ou diferentes, podemos utilizar == ou ! = (junto).
  • veja um exemplo:
    • @debug 100px ≠ 10rem // true
    • @debug “Helvetica” == Helvetica // true
  • Os resultados de operações são sempre booleanos.

Maior ou Menor

  • Caso haja a necessidade de comprar valores, se seão MAIORES OU MENORES, podemos utilizar < ou >.
  • Veja um exemplo:
    • @debug 100 > 20 // true
  • Os resultados de operações são sempre BOOLEANOS.
  • Podemos adicionar um igual para rtambém verificar a iguldade, ex: >=

  • Tome cuidado com a compatibilidade de valores.
