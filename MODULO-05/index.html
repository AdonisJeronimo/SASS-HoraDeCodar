Partials

Os partials são uma forma de dividir o código CSS em arquivos menores e mais organizados. Ela envolve o uso de arquivos parciais (partials) que contêm trechos de código CSS ou SASS reutilizáveis e, em seguida, são importados em um arquivo principal para compilação.

Para criar um arquivo parcial, você precisa nomeá-lo com um prefixo de _ (underline) no início do nome do arquivo, por exemplo _variables.scss. Essa convenção de nomenclatura é uma convenção comum para identificar arquivos que não são os principais do projeto, sendo ele um partials que irá ter uma inclusão em outro arquivo. Ao importar um arquivo parcial em um arquivo principal do SASS, você usa a diretiva @import. Aqui está um exemplo:

@import 'variables'

Mixins

É uma funcionalidade que nos permite agrupar um conjunto de propriedades CSS em um bloco de código reutilizável. Com mixins, você pode definir estilos complexos ou repetitivos uma vez e usá-los em vários lugares do seu código. Para criar um mixin no SASS, você usa a diretiva @mixin seguida de um nome para o mixin e um bloco de código contendo as propriedades CSS desejadas. Aqui está um exemplo simples:

@mixin efeito-hover propriedade

Mixins com parâmetros

Os mixins no SASS também podem receber parâmetros, permitindo torná-los ainda mais flexíveis e dinâmicos. Os parâmetros são definidos entre parênteses ao declarar o mixin e podem ser usados dentro do bloco de código do mixin. Caso passemos mais de um parâmetro, separamo-los por vírgulas. EXEMPLO:

Parâmetros opcionais

No SASS, você pode definir argumentos opcionais em mixins usando a sintaxe de atribuição de valor padrão aos parâmetros. Isso permite que você chame um mixin com ou sem fornecer um valor para o argumento opcional.

Por exemplo, se definirmos criarmos um mixin com nome de custom-border e passarmos os seguintes paramentros, $style,$color,$width:1px, estamos dizendo que caso nao seja passado nenhum valro ao parametro de largura, automaticamente ele usara seu valor opcional

Vale resaltar que por regra, os argumentos opcionais devem ficar depois dos obrigatorios.

Herança

As heranças no SASS referem-se à capacidade de um seletor herdar as propriedades de outro seletor. Isso permite que você reutilize estilos e evite a repetição de código, facilitando a manutenção e a criação de estilos consistentes.

No SASS, a herança é feita usando o operador @extend. Com esse operador, você pode fazer com que um seletor herde todas as propriedades de outro seletor.

Calculos com SASS

O SASS oferece suporte a cálculos matemáticos básicos, permitindo que você execute operações aritméticas dentro do seu código SASS. Isso pode ser útil para calcular valores de propriedades CSS, como largura, altura, margem, etc.

Funções

No SASS, você pode criar e usar funções para realizar cálculos, manipular valores e gerar estilos dinamicamente. As funções no SASS são semelhantes às funções em outras linguagens de programação e podem ser bastante úteis para criar código reutilizável.

Sintaxe: @function nome(argumentos)

Criando erros no SASS

Podemos gerar um erro com o SASS. Uma situação interessante é: quando criamos uma função e ela precisa de alguns valores mínimos para argumentos, então caso o programador não insira corretamente isso fará com que dispare um erro com o @error. Podemos interpolar valores na mensagem também.

Gerando um warning

Podemos gerar um aviso com o SASS. A situação pode ser a mesma que a do erro, mas este método é menos invasivo, ele não para a alicação, apenas mostra no console. Então caso o programador não insira corretamente, disparamos um erro com o @warn.

Depuração de valores

No SASS, você pode usar a diretiva @debug para imprimir informações de depuração no console durante a compilação. A diretiva @debug é útil para exibir valores de variáveis, mensagens informativas ou qualquer outra informação relevante que possa ajudar na depuração e solução de problemas.

Estruturas condicionais

Podemos criar uma estrutura de condição no SASS, utilizando o @if e o @else, assim como também podemos usar o @else if. Estas regras funcionam como em outras linguagens de programação.

Each no SASS

Podemos criar um estrutura de repetição em listas no SASS, utilizando o `@each` . Podemos repetir uma criação de calsse n vezes, por exemplo, veja a sintaxe(onde $size é a lista de valores): `@each $size in $size`

For no SASS

Podemos criar um estrutura de repetição em listas no SASS, utilizando o `@for`. Podemos repetir quantas vezes for necessario, por medio de condições. por exemplo, veja a sintaxe: `@for $i from 1 through 3` (repetir de 1 até chegar a 3)
